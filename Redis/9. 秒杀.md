# 优惠券秒杀

### 1.1 全局唯一ID

#### 1.1.1 背景

1. 优惠券是黑马点评项目中唯一的商品，其表结构如下图所示，voucher_id代表购买的优惠券的id，我们需要一个字段来表示订单编号，传统方法是自增主键id+订单id，但是考虑到之后数据量大之后，要进行分库分表操作，这样主键id就可能在多个表中重复，这样会让我们的订单的唯一性被破坏。

   ![image-20260406201455591](https://gcore.jsdelivr.net/gh/weiqiang612/My-TuChuang@main/img/CangQiong/image-20260406201455591.png)

2. 我们使用全局唯一id来作为主键，这里使用了一种类雪花算法，它的结构如下图所示，而这种ID也是对数据库的B+树索引友好的，如果基于等值查询(id = #{id})，和原来的自增主键id的性能是一样的，如果基于范围查询（a <= id <= b），由于其时间戳的原因，临近时间生成的ID，在B+树上的位置也是比较近的，不会落后自增id多少。

   ![image-20260406202126075](https://gcore.jsdelivr.net/gh/weiqiang612/My-TuChuang@main/img/CangQiong/image-20260406202126075.png)

#### 1.1.2 概念

全局ID生成器，是一种在分布式系统中用来生成全局唯一ID的工具，一般要满足一下特性：

* 唯一性
* 高可用
* 高性能
* 递增性（用于B+树索引）
* 安全性

这里我们使用Redis的 INCR 功能做全局ID生成器，其非常满足高可用和高性能的特点
