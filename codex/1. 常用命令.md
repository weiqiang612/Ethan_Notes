# Codex 常用 `/` 命令

适用范围：交互式 Codex CLI 会话中的斜杠命令，不是终端里直接执行的 `codex xxx` 子命令。

本清单按我最常用的场景梳理，基于当前环境核对到的版本：`codex-cli 0.117.0`。

## 先看最常用的一组

`/status`
- 查看当前会话状态。
- 常用来确认模型、审批策略、可写目录、上下文占用。

`/model`
- 切换当前会话使用的模型。
- 任务复杂时切强模型，轻任务时切轻量模型。

`/permissions`
- 调整当前会话的权限/审批策略。
- 适合在“先保守查看”与“允许自动执行”之间切换。

`/compact`
- 压缩并总结当前长对话，释放上下文。
- 会话很长、响应开始变慢时很好用。

`/diff`
- 查看当前工作区的改动，包括未跟踪文件。
- 适合在提交前快速复核。

`/review`
- 让 Codex 直接审查当前工作区改动。
- 适合在改完代码后做一轮风险检查。

`/clear`
- 清屏并开始一段新的对话。
- 想“重新开聊”时用它，不只是清屏。

`/new`
- 在同一个 CLI 会话里新开一个对话。
- 和 `/clear` 不同，它不会先清空当前终端视图。

`/resume`
- 恢复以前保存过的会话。

`/fork`
- 基于当前对话分叉出一个新线程。
- 适合保留原方案，同时试另一种实现。

`/mention`
- 把文件或目录明确附加到当前对话。
- 例如：`/mention src/app.ts`

`/init`
- 在当前目录生成 `AGENTS.md` 模板。
- 适合给仓库补长期规则。

`/mcp`
- 查看已配置的 MCP 工具。

`/apps`
- 浏览可接入的 app/connectors，并把它们插入到当前提示词里。

`/copy`
- 复制最近一次已完成的 Codex 输出。

`/quit`
- 退出当前 CLI。
- `/exit` 与它等价。

## 按场景查命令

### 1. 看状态 / 调环境

`/status`：看当前会话配置和 token 使用情况。  
`/model`：切模型。  
`/fast`：切换或查看 Fast mode。  
`/permissions`：改审批/权限策略。  
`/debug-config`：排查配置来源和生效层级。  
`/statusline`：配置底部状态栏显示项。

### 2. 管理上下文 / 会话

`/compact`：压缩上下文。  
`/clear`：清屏并新开对话。  
`/new`：新开对话但不先清屏。  
`/resume`：恢复历史会话。  
`/fork`：从当前对话分叉新线程。  
`/agent`：切换到某个 agent 线程。

### 3. 看改动 / 做代码工作

`/diff`：查看 Git 改动。  
`/review`：让 Codex 做代码审查。  
`/mention`：显式附加文件/目录。  
`/init`：生成 `AGENTS.md`。

### 4. 工具 / 扩展能力

`/mcp`：列出 MCP 工具。  
`/apps`：浏览并插入 app。  
`/experimental`：切换实验性功能。  
`/ps`：查看实验性的后台终端及最近输出。

### 5. 交互 / 退出

`/copy`：复制最近输出。  
`/personality`：调整回答风格。  
`/plan`：切到 plan 模式，并可直接附带一个规划请求。  
`/quit`：退出。  
`/exit`：退出，等同于 `/quit`。  
`/logout`：退出登录。  
`/feedback`：提交日志/反馈。

## 补充说明

- `/approvals` 仍可作为别名使用，但官方文档说明它已经不再出现在 slash 弹出列表里。
- 如果你忘了命令名，最稳妥的做法是在交互界面里直接输入 `/`，先看当前版本实际弹出的命令列表。
- 这里写的是“交互态 `/` 命令”；像 `codex exec`、`codex review`、`codex resume` 这类属于终端子命令，不在本文件范围内。
