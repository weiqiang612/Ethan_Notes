# 进程管理

>
>
>1. 什么是进程控制块PCB？进程三种状态运行、阻塞、就绪间转换的原因是什么？
>   * 进程控制块是一个由操作系统维护的、用于记录进程相关信息的数据结构
>   * 就绪->运行：处理器从就绪进程中选择一个进程进入运行态
>   * 运行->就绪：时间片用完或高优先级进程就绪等使当前进程暂停运行
>   * 运行->阻塞：进程请求事件未出现而进入阻塞状态
>   * 阻塞->就绪：进程请求事件出现
>2. 关于创建状态的进程，创建工作主要有哪些？
>   * 申请空白PCB
>   * 为新进程分配其运行所需的资源，包括各种物理和逻辑资源
>   * 初始化进程控制块PCB
>   * 如果进程就绪队列能够接纳新进程，便将新进程插入就绪队列
>3. 什么是进程间的互斥和同步
>   * 进程互斥：指由于共享资源要求排他性使用，进程间要相互竞争以使用这些互斥资源
>   * 进程同步：指多个进程中发生的事件存在时序关系，必须协同动作相互配合共同完成一个任务
