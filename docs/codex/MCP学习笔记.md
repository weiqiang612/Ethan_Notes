# MCP学习笔记

## MCP 是什么

MCP（Model Context Protocol，模型上下文协议）是一个开源标准，用来把 AI 应用连接到外部系统。

可以把它理解成 AI 世界里的“USB-C 接口”：统一连接方式，让模型更容易接入各种数据源、工具和工作流。

## 核心概念

- `Client`：发起连接的 AI 应用或助手，例如聊天应用、IDE、Agent。
- `Server`：向外提供能力的服务，负责暴露数据和工具。
- `Data sources`：外部数据源，例如本地文件、数据库、日历、笔记系统。
- `Tools`：可调用的功能，例如搜索、计算、执行操作。
- `Workflows`：可复用的工作流或专用提示，让模型执行更具体的任务。

## MCP 能做什么

- 让 AI 访问个人或企业数据。
- 让 AI 调用外部工具完成任务。
- 让 AI 直接接入现有应用生态，而不是为每个系统单独做集成。

## 为什么重要

- 对开发者：减少集成成本和开发复杂度。
- 对 AI 应用：扩展能力边界，提升可用性。
- 对用户：让 AI 更懂上下文，也能更主动地执行操作。

## 一句话总结

MCP 的目标是提供一种通用、标准化的方式，让 AI 应用安全、可扩展地连接外部数据和工具。
